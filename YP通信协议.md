# YP 设备串口通信协议

> 适用固件：Ver 1.0（以 `<A1>` 查询结果为准）  
> 文档依据当前 `main.c` / `main.h` 源码整理，供使用说明与上位机开发参考。

---

## 1. 物理层参数

| 项目 | 参数 |
|------|------|
| 接口 | UART1 |
| MCU 引脚 | TX = P31，RX = P30 |
| 波特率 | **115200** |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |

---

## 2. 帧格式

### 2.1 主机 → 设备（命令帧）

```
<>命令\r\n
```

| 字段 | 说明 |
|------|------|
| `<>` | 固定帧头，必须位于命令开头 |
| 命令 | ASCII 字符串，见下文命令表 |
| `\r\n` | 帧结束符（回车 + 换行） |

**示例**

```
<>A5\r\n
<>M05\r\n
<>C1=1\r\n
```

### 2..2 帧接收机制

- 设备按字节接收，以 `\r\n` 作为一帧结束。
- 收到新字节后启动 **48ms** 超时计时（3 × 16ms 串口任务周期）；超时后解析缓冲区内容。
- 接收缓冲区最大 **64 字节**，超出后计数归零。
- 帧头不是 `<>` 的命令将被忽略（无应答）。

### 2.3 设备 → 主机（应答/上报）

- 文本格式，UTF-8 兼容 ASCII。
- 每条消息以 `\r\n` 结尾。
- 查询类命令返回对应数据行；控制类命令返回 `OK` 或 `ERR`。
- 部分状态变化由设备**主动上报**，无需主机轮询。

## 3. 命令一览

### 3.1 查询命令（A 类）

| 命令 | 功能 | 应答格式 | 示例 |
|------|------|----------|------|
| `<>A1` | 固件版本 | `Ver=主.次\r\n` | `Ver=1.0\r\n` |
| `<>A2` | 板载温度 | `Temp=温度\r\n` | `Temp=25.3\r\n` |
| `<>A3` | 电池电压 | `Vref=电压\r\n` | `Vref=3.52\r\n` |
| `<>A4` | 当前出光模式编号 | `Mode=编号\r\n` | `Mode=5\r\n` |
| `<>A5` | 完整运行状态 | 见 [4.1 Sta 状态帧](#41-sta-完整状态) | `Sta=0,0,0,0,0,0,0\r\n` |

> 温度单位：℃；保留 1 位小数。  
> 电压单位：V；保留 2 位小数。

---

### 3.2 模式控制（M 类）

| 命令 | 功能 | 应答 |
|------|------|------|
| `<>M00` | 全部关闭 | `OK\r\n` |
| `<>M01` ~ `<>M13` | 切换到指定出光模式 | `OK\r\n` 或 `ERR\r\n` |

#### 出光模式编号表

| 编号 | 命令 | 模式名称 | 说明 |
|------|------|----------|------|
| 0 | `<>M00` | 全部关闭 | 关闭所有光源 |
| 1 | `<>M01` | 激光常亮 | |
| 2 | `<>M02` | 激光瞬亮 | 串口仅设置模式编号，瞬亮需按键触发 |
| 3 | `<>M03` | 激光 + 白光常亮 | |
| 4 | `<>M04` | 激光 + 白光瞬亮 | 串口仅设置模式编号 |
| 5 | `<>M05` | 白光常亮 | |
| 6 | `<>M06` | 白光瞬亮 | 串口仅设置模式编号 |
| 7 | `<>M07` | 白光 8Hz 眩目常亮 | 含爆闪 |
| 8 | `<>M08` | 白光 8Hz 眩目瞬亮 | 串口仅设置模式编号 |
| 9 | `<>M09` | 弱白光常亮 | |
| 10 | `<>M10` | 红光瞬亮 | 串口仅设置模式编号 |
| 11 | `<>M11` | 红光常亮 | |
| 12 | `<>M12` | 绿光瞬亮 | 串口仅设置模式编号 |
| 13 | `<>M13` | 绿光常亮 | |

> **说明**：`M01`、`M03`、`M05`、`M07`、`M09`、`M11`、`M13` 可通过串口直接开启对应常亮输出。  
> 编号 2、4、6、8、10、12 为瞬亮类模式，主要由按键手势触发；串口写入后会更新模式编号，但不会自动维持瞬亮输出逻辑。

---

### 3.3 快捷控制（B 类）

| 命令 | 功能 | 应答 |
|------|------|------|
| `<>B0` | 关闭全部输出（等同 M00） | `OK\r\n` |

---

### 3.4 通道独立控制（C 类）

格式：`<>Cx=n`，其中 `x` 为通道号，`n` 为 `0`（关）或 `1`（开）。

| 命令 | 通道 | 开启后关联模式 |
|------|------|----------------|
| `<>C1=0` / `<>C1=1` | 激光 | `MODE_LASER_HOLD`（1） |
| `<>C2=0` / `<>C2=1` | 白光（强） | `MODE_WHITE_HOLD`（5） |
| `<>C3=0` / `<>C3=1` | 白光（弱） | `MODE_WHITE_DIM_HOLD`（9） |
| `<>C4=0` / `<>C4=1` | 绿光 | `MODE_GREEN_HOLD`（13） |
| `<>C5=0` / `<>C5=1` | 红光 | `MODE_RED_HOLD`（11） |

- 关闭某通道后，若所有光源均已关闭，模式自动归零（`MODE_OFF`）。
- 应答：`OK\r\n` 或 `ERR\r\n`（通道号非法）。

---

### 3.5 爆闪控制（D 类）

| 命令 | 功能 | 应答 |
|------|------|------|
| `<>D1=0` | 关闭 8Hz 爆闪 | `OK\r\n` |
| `<>D1=1` | 开启 8Hz 爆闪 | `OK\r\n` 或 `ERR\r\n` |

- 爆闪频率约 **8Hz**（周期 125ms）。
- 若开启爆闪时白光未开，设备会自动将白光设为强光输出。

---

### 3.6 系统控制

| 命令 | 功能 | 应答 |
|------|------|------|
| `<>REBOOT` | 软件复位重启 | 无（设备立即复位） |

---

## 4. 设备主动上报

### 4.1 Sta 完整状态

当模式、档位或任一光源输出发生变化时，设备自动发送（周期检测 20ms）：

```
Sta=mode,gear,v5,white,green,red,laser\r\n
```

| 字段 | 含义 | 取值 |
|------|------|------|
| mode | 出光模式编号 | 0 ~ 13，见 [3.2 模式表](#32-模式控制m-类) |
| gear | 当前档位 | 0=无，1=左档，2=中档，3=右档 |
| v5 | 5V 升压使能 | 0=关，1=开 |
| white | 白光输出 | 0=关，1=强光，2=弱光 |
| green | 绿光输出 | 0=关，1=开 |
| red | 红光输出 | 0=关，1=开 |
| laser | 激光输出 | 0=关，1=开 |

**示例**

```
Sta=5,3,1,1,0,0,0\r\n
```

表示：模式 5（白光常亮）、右档、5V 开、白光强光、绿/红/激光均关。

---

### 4.2 生命周期消息

| 消息 | 触发时机 |
|------|----------|
| `RUN!\r\n` | 上电启动或休眠唤醒复位后 |
| `Shutdown\r\n` | 中档保持 5 秒进入休眠前 |

---

### 4.3 报警消息

| 消息 | 触发条件 | 说明 |
|------|----------|------|
| `Alarm=LowVolt\r\n` | 电池电压 < 3.0V 持续约 1s | 仅上报一次，直至电压 > 3.1V 后恢复 |

> 低压报警不会自动关机。  
> 电压低于 2.6V 时，若当前无任何出光，则禁止通过串口或按键**新开启**模式（见 [5. 低压保护](#5-低压保护)）。

---

## 5. 低压保护

| 阈值 | 行为 |
|------|------|
| < 3.0V | 串口上报 `Alarm=LowVolt` |
| > 3.1V | 低压报警标志清除 |
| < 2.6V 且当前完全关闭 | 禁止开启新模式/通道（`M01~M13`、`C1~C5=1`、`D1=1` 返回 `ERR`） |
| 已有出光模式中 | 不检测 2.6V 限制，可继续使用直至手动关闭 |

**允许的操作（低压且已全关时）**

- 查询命令 `A1` ~ `A5`
- 关闭命令 `M00`、`B0`、`Cx=0`、`D1=0`

---

## 6. 控制应答说明

| 应答 | 含义 |
|------|------|
| `OK\r\n` | 命令执行成功 |
| `ERR\r\n` | 命令格式错误、参数非法，或低压禁止开启 |

无效帧（帧头不是 `<>`）无应答。

## 7. 通信示例

### 7.1 连接后读取状态

```
主机 →  <>A5\r\n
设备 ←  Sta=0,0,0,0,0,0,0\r\n

主机 →  <>A1\r\n
设备 ←  Ver=1.0\r\n

主机 →  <>A2\r\n
设备 ←  Temp=26.5\r\n

主机 →  <>A3\r\n
设备 ←  Vref=3.48\r\n
```

### 7.2 开启白光常亮

```
主机 →  <>M05\r\n
设备 ←  OK\r\n
设备 ←  Sta=5,0,1,1,0,0,0\r\n
```

### 7.3 独立控制激光

```
主机 →  <>C1=1\r\n
设备 ←  OK\r\n
设备 ←  Sta=1,0,1,0,0,0,1\r\n

主机 →  <>C1=0\r\n
设备 ←  OK\r\n
设备 ←  Sta=0,0,0,0,0,0,0\r\n
```

### 7.4 低压禁止开启

```
主机 →  <>M05\r\n
设备 ←  ERR\r\n
（此时 Vref < 2.6V 且设备处于全关状态）
```

### 7.5 设备启动与休眠

```
设备 ←  RUN!\r\n
（上电或唤醒）

… 中档保持 5 秒 …

设备 ←  Shutdown\r\n
（进入休眠，P32 边沿唤醒后再次 RUN!）
```

## 8. 档位与按键（非串口，供对照）

串口协议本身不控制档位切换，但 `Sta` 帧中的 `gear` 字段会反映物理档位：

| gear | 档位 | 引脚 |
|------|------|------|
| 0 | 无 | — |
| 1 | 左档 | P34 |
| 2 | 中档 | P32 |
| 3 | 右档 | P33 |

档位变化时设备会自动关闭全部输出。

---

## 9. 上位机开发建议

1. 连接后先发送 `<>A5` 同步完整状态。
2. 控制命令发送后，可等待 `OK`/`ERR`，也可监听自动上报的 `Sta=` 确认结果。
3. 建议订阅 `Sta=`、`RUN!`、`Shutdown`、`Alarm=LowVolt` 做 UI 状态刷新。
4. 收到 `ERR` 时，可查询 `<>A3` 判断是否因低压保护导致。
5. Web 调试工具参考项目内 `serial_tool.html`（Chrome / Edge，Web Serial API）。
